Despite appearing fairly innocuous, loading data can be one of the most frustrating challenges when starting a data science project. Missing values, trailing commas, superfluous quotes in strings and many other issues can all cause headaches when reading a csv file. I’m sure many an aspiring data scientist has been left feeling discouraged after falling at this early hurdle - I know I have.

This post introduces some basics of loading data using a csv. It assumes that you’ve managed to get your data into a reasonable format without any of the above issues. I recommend viewing your csv in a text editor such as Notepad++ or Atom before attempting any coding. Be wary of excel as it can hide (and sometimes introduce) problems with csv files.

Packages

This tutorial will use R’s readr package. This package contains the read_csv function which is similar to base R’s read.csv function, but with some enhancements.

Load the data

Using readr

If you haven’t already, install the readr package by running

install.packages("readr")

Data can be loaded with the code below. Here I have my data stored in a folder called “data” in the current working directory. I recommend structuring code so that paths relative to your current working directory are used. That way, if you ever need to move your project to another computer or directory, you can simply copy the entire project directory and still have working code. Code may fail if absolute paths are used and any part of a project is moved.

require(readr)
csv_data <- read_csv(file = "data/2017-10-17-data.csv")
## Parsed with column specification:
## cols(
##   Date = col_character(),
##   Project = col_character(),
##   Component = col_character(),
##   Income = col_integer()

One nice thing about read_csv compared to read.csv is data is imported as a tibble rather than a data.frame. Tibbles have a few nice features such as only showing the top few rows which is handy if you are dealing with a lot of data. I’ve accidentally filled my console by forgetting to print head(csv_data) many times. Another nice touch is character columns are imported as characters rather than factors by default. Factors can be useful, but they are also fiddly to deal with especially for newcomers to R.

Some extras

Column type specification

One problem with our imported data is the Date column has been read as a string. Conveniently, readr allows us to specify the data type of each column using the col_types argument and cols function. A warning will be output if any of the dates are in an invalid format.

csv_data <- read_csv(file = "data/2017-10-17-data.csv",
                     col_types = cols(
                       Date = col_date(format = "%d/%m/%Y"),
                       Project = col_character(),
                       Component = col_character(),
                       Income = col_integer()
                       )
                     )

Remembering the correct date specification is a real pain (this can be useful). Personally I find it easier to use the lubridate and dplyr packages to convert strings into dates, but that’s a topic for another tutorial!

White space trimming

Our made up data contains a few entries with leading or trailing whitespace characters. By default, read_csv will remove any trailing or leading whitespace characters, but this behaviour can be disabled by setting trim_ws = FALSE.
