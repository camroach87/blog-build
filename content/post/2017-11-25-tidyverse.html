This series of blog posts give a brief introduction to R’s tidyverse. The tidyverse is a fantastic framework for shaping, exploring and visualising data. Consistent syntax, intuitive behaviour and excellent performance are the main reasons to learn this set of tools. Furthermore, tidyverse packages also have stickers that may be assembled in a geometrically pleasing hex formation.

Figure 1: I’ll never stop.

The hope is that by breaking the key packages into 5 minute tutorials a beginner should quickly pick up the main capabilities of each. Many excellent resources detailing the finer points of tidyverse already exist (see R for Data Science and the tidyverse documentation) and so this series will be quite shallow by comparison. However, I believe there is a lot a beginner may imbibe from such a brief introduction. Results first, details later is the guiding principle here.

The series will be broken into 5 tutorials focusing on:

  1. Reading data into R using readr and tibble.
  2. Reshaping data into long or wide data formats with tidyr.
  3. Manipulating or summarising data columns using dplyr.
  4. Plotting results with ggplot2.
  5. (Maybe) doing some modelling with purrr/broom/modelr/validatr.

I say maybe because modelling with tidyverse isn’t quite as easy as the other tasks. I’ve had a go at trying to improve this (validatr), but that still needs a lot of work. I’m not sure if I’m capable of producing a 5-minute tutorial on modelling with tidyverse and so it might be best left to a longer more detailed post.

My hope is that by the end of this series, somebody with very little R programming experience should be able to look at the following code chunks and know exactly what is going on.

read_csv("./data/5-min-tidyverse-data.csv") %>% 
  group_by(Project) %>% 
  summarise(Income = sum(Income)) %>% 
  filter(!is.na(Project)) %>% 
  ggplot(aes(x = Project, y = Income, fill = Project)) +
  geom_col() + 
  labs(title = "Income across projects",
       subtitle = "Using a daft data set") +
  theme_bw() +
  theme(legend.position = "none")

read_csv("./data/5-min-tidyverse-data.csv") %>% 
  mutate(Date = dmy(Date),
         Year = year(Date)) %>% 
  filter(!is.na(Project),
         !is.na(Year)) %>% 
  group_by(Project, Year) %>% 
  summarise(Income = mean(Income)) %>% 
  select(Project, Year, Income) %>% 
  spread(Project, Income) %>% 
  kable(caption = "Average income per component for important business projects.")
